4. The client executes a DNS search for SRV records that match the returned Autodiscover server
URI.
5. If the DNS server responds with any SRV records, the corresponding Autodiscover server URI
records are added to the list of possible Autodiscover server URIs on the client. It is not an error
if the DNS server does not return any DNS SRV records in response to the DNS search.
6. The client uses the Autodiscover server URI to contact the Autodiscover server via HTTP to query
server information.
